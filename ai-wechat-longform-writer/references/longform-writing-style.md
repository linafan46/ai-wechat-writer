# 长文叙事与排版

## 核心定位

这不是新闻长文，也不是教程，而是一篇“用最新事实解释普通人变化，并给读者一个具体收获”的公众号中长文。

文章要让读者感觉：

```text
我以前知道这件事，但没这样理解过，而且我知道今天能怎么做了。
```

## 推荐结构

优先使用六段式：

1. **具体场景**：从读者熟悉的工作、学习或生活画面开始。
2. **指出疲惫或困惑**：把问题说清楚，但不要急着上结论。
3. **引入事实信号**：引用最新 AI 热点或官方发布，复杂名词要降噪。
4. **具体案例**：用职业规划、内容创作、学习、资料整理等普通人能代入的案例解释。
5. **落到可执行动作**：说明读者今天能怎么整理、怎么问、怎么给 AI 上下文。
6. **回扣开头**：回到最开始的场景，给出一句可传播判断。

## 长度和密度

默认写 700～1200 字。宁可短一点，也不要把一个观点反复说三遍。

删除这些内容：

- 对热点功能的长篇复述。
- 用不同说法重复同一个观点。
- 太多“为什么重要”的解释，但没有告诉读者怎么做。
- 过长的清单和模板。

保留这些内容：

- 一个具体场景。
- 一个事实信号。
- 一个能让读者看懂的案例。
- 一个今天就能用的动作。
- 一句能传播的结尾判断。

## 黑底白字分割卡

长文需要用黑底白字分割卡降低阅读压力。它不是“小结”，也不是装饰，而是章节标题和逻辑路标。

分割卡适合放在这些位置：

- 正式进入核心事实或原文主体之前。
- 从外部证据切到内部证据之前。
- 从事实分析切到类比解释之前。
- 从类比切回个人启发之前。
- 从风险讨论切到行动建议之前。

分割卡文案要短，通常 8～18 个字。可以是一句判断，也可以是一个问题。不要写成长解释。

好：

```text
真正的效率，不是替换工具
```

```text
来自外部世界的证据
```

```text
如果我们错了呢？
```

```text
AI 还没进入你的工作现场
```

不好：

```text
小结
```

```text
这一部分主要讲的是为什么 AI 的价值不只是模型更聪明，而是进入工作系统。
```

如果文章是解读长文或报告，分割卡可以更像原文章节标题，例如：

```text
当 AI 开始构建自己

来自外部世界的证据

来自 Anthropic 内部的证据

未来工作可能是什么样？

如果我们错了呢？

可能的未来

我们应该怎么做？
```

如果文章是观点长文，分割卡要更像观点路标，并尽量短，例如：

```text
你不是不会用 AI

AI 还没进入工作现场

真正的效率，不是替换工具

未来差距在工作系统
```

交付正文时，用下面格式标注，方便用户后续排版成黑底白字卡片：

```markdown
【黑底白字分割卡】
真正的效率，不是替换工具
```

## 叙事节奏

长文不是短句堆叠。段落要比短文更完整，通常 2～5 句一段。

可以单独成段的句子：

```text
所以你会累。

但方向是对的。

这才像一个助理。
```

这些句子必须承担转折、停顿或强调作用。普通解释句不要单独拆开。

## 实操写法

每篇文章必须让读者带走一个具体技能、方法或动作。

但实操不能写成“表格说明书”，也不要把十几个字段铺成清单。更好的写法是像跟朋友聊天，把方法揉进一个场景里。

好：

```text
比如你想让 AI 帮你做职业规划，最不该问的是，我适合做什么。

这句话太空了。

你不如先给它一段更像真人的背景，比如自己在哪个城市，做什么工作，学历和专业是什么，工作几年了，最近最想解决的是涨薪、跳槽、转行，还是单纯想把日子过得没那么拧巴。

再补一点更关键的东西，你擅长什么，不擅长什么，每天大概能拿出多少时间，哪些工作方式你接受不了，以及你未来一年、三年，大概想把自己挪到什么位置。

这些东西不用写得很正式，也不用像填简历，你就当是在给一个靠谱朋友介绍自己。
```

不好：

```text
你的职业是什么。

你的学历和专业背景是什么。

你现在的工作内容是什么。

你擅长什么，不擅长什么。
```

要点：

- 实操要有，但不要密集罗列。
- 清单最多保留 3 个核心词，例如背景、目标、案例。
- 如果必须给模板，模板要短，只给骨架，不写成长表。
- 每个热点都要回答，读者看完能学会什么。

## 加粗规则

长文必须有加粗重点，用来帮助没耐心的读者扫读。加粗不是装饰，而是正文里的“抓手”。

通常一篇 900～1800 字长文，需要 5～8 处加粗。加粗优先给这些内容：

- 文章核心判断。
- 段落关键转折。
- 类比之后得出的结论。
- 个人启发或行动判断。
- 结尾可传播句。

不要加粗普通事实、产品名、资料来源，也不要整段加粗。

好：

```markdown
**AI 的下一步，不是更会聊天，而是进入系统。**
```

```markdown
**它只是让旧流程跑得快了一点。**
```

```markdown
**未来人与 AI 的差距，可能不会只体现在谁会写提示词，而是谁更早把自己的工作，整理成了一个能被 AI 理解、参与、改进的系统。**
```

不好：

```markdown
**微软 Build 讲了 Microsoft IQ、Work IQ、Scout、Agent 365、Foundry。**
```

```markdown
**整段很长很长的解释全部加粗，让读者不知道重点在哪里。**
```

## 标点与连贯性

标点要服务阅读节奏。长文不要为了显得有力，把一段话切成很多短句号。

同一层意思，优先用逗号、顿号、分号连接，让段落读起来连贯。只有转折、停顿、强调和段落结束时，才用句号。

好：

```text
复制完之后，还要继续补充，这个项目之前有个限制，这个客户不喜欢太激进的方案，这个老板只看结论，不看过程。
```

不好：

```text
复制完之后，还要继续补充。这个项目之前有个限制。这个客户不喜欢太激进的方案。这个老板只看结论，不看过程。
```

好：

```text
真正的效率提升，是后来大家重新设计工厂，机器怎么摆，工序怎么流，工人怎么协作，整个生产方式都被电力重新组织了一遍。
```

不好：

```text
真正的效率提升，是后来大家重新设计工厂。机器怎么摆。工序怎么流。工人怎么协作。整个生产方式都被电力重新组织了一遍。
```

检查方法：

- 如果连续 3 句都很短，先判断它们是否属于同一层意思；如果是，合并。
- 如果一段里句号太密，改用逗号、顿号或分号恢复语气。
- 保留少数短句，但必须承担强调作用，例如“所以你会累。”“但方向是对的。”

## 语言要求

- 语气直白、克制、有个人判断。
- 可以使用“我觉得”“坦率地讲”“你想想看”，但不要过度口语化。
- 热点事实只保留支撑观点所需的部分，不复述发布会。
- 复杂产品名出现一次即可，之后翻译成普通话。
- 多用“不是……而是……”“短期看……长期看……”“以前……以后……”推动观点。
- 避免“首先、其次、最后”“综上所述”“赋能、抓手、闭环”等套话。
- 不写夸张预测，不把个人推断说成事实。

## 类比写法

类比要承担解释任务，不是装饰。

好类比的结构：

```text
先写历史或生活里的具体情况。
再指出它和 AI 现状的相似处。
最后回到文章主观点。
```

示例方向：

- 电力进入工厂：效率提升不是替换动力，而是重建工厂布局、流程和协作。
- 智能手机前夜：爆发不只来自芯片变强，也来自流量、App 和生态成熟。
- 图书馆索引：资料不整理，再聪明的人也只能临时翻找。
- 助理进办公室：真正有用的助理不是会说话，而是知道资料、流程和边界。

## 推荐排版

标题、摘要、正文分开交付。

正文里不要使用很多普通小标题。需要明确切分长文时，使用黑底白字分割卡。分割卡数量通常 2～5 个，长报告解读可以更多，但必须跟文章逻辑一致。

默认版式：

```text
正文 16px
正文行高 1.5
黑底白字分割卡 20px
分割卡行高 1.5
```

连续解释要合并成段，关键判断再单独成段。示例：

```markdown
你的工作不在一个聊天框里。你的工作散落在文档、表格、会议、日程、聊天记录和过去无数次判断里。谁说过什么，项目卡在哪里，客户为什么不满意，哪些事能交给 AI，哪些事必须自己确认，这些才是工作现场。

但今天大多数人用 AI，还是每次重新把这个现场搭一遍。搭完一次，拆掉一次。下次再问，再从头开始。

所以你会累。
```

## 结尾要求

结尾必须回扣开头场景，不要只总结全文。

结尾句要像观点，而不是口号。示例：

```text
AI 的下一步，不是更会聊天，而是进入你的工作现场。
```

```text
未来人与 AI 的差距，可能不是谁更会提问，而是谁留下了更多值得被理解的自己。
```
