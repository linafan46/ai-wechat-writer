# 长文叙事与排版

## 核心定位

这不是新闻长文，也不是教程，而是一篇“用最新事实解释普通人变化”的公众号中长文。

文章要让读者感觉：

```text
我以前知道这件事，但没这样理解过。
```

## 推荐结构

优先使用六段式：

1. **具体场景**：从读者熟悉的工作、学习或生活画面开始。
2. **指出疲惫或困惑**：把问题说清楚，但不要急着上结论。
3. **引入事实信号**：引用最新 AI 热点或官方发布，复杂名词要降噪。
4. **直观类比**：用工厂、电力、手机、地图、图书馆、助手等类比降低理解成本。
5. **落到个人启发**：说明普通人今天能怎么理解、整理或调整。
6. **回扣开头**：回到最开始的场景，给出一句可传播判断。

## 叙事节奏

长文不是短句堆叠。段落要比短文更完整，通常 2～5 句一段。

可以单独成段的句子：

```text
所以你会累。

但方向是对的。

这才像一个助理。
```

这些句子必须承担转折、停顿或强调作用。普通解释句不要单独拆开。

## 语言要求

- 语气直白、克制、有个人判断。
- 可以使用“我觉得”“坦率地讲”“你想想看”，但不要过度口语化。
- 热点事实只保留支撑观点所需的部分，不复述发布会。
- 复杂产品名出现一次即可，之后翻译成普通话。
- 多用“不是……而是……”“短期看……长期看……”“以前……以后……”推动观点。
- 避免“首先、其次、最后”“综上所述”“赋能、抓手、闭环”等套话。
- 不写夸张预测，不把个人推断说成事实。

## 类比写法

类比要承担解释任务，不是装饰。

好类比的结构：

```text
先写历史或生活里的具体情况。
再指出它和 AI 现状的相似处。
最后回到文章主观点。
```

示例方向：

- 电力进入工厂：效率提升不是替换动力，而是重建工厂布局、流程和协作。
- 智能手机前夜：爆发不只来自芯片变强，也来自流量、App 和生态成熟。
- 图书馆索引：资料不整理，再聪明的人也只能临时翻找。
- 助理进办公室：真正有用的助理不是会说话，而是知道资料、流程和边界。

## 推荐排版

标题、摘要、正文分开交付。

正文里不要使用很多小标题。需要转场时，用自然段推进。

连续解释要合并成段，关键判断再单独成段。示例：

```markdown
你的工作不在一个聊天框里。你的工作散落在文档、表格、会议、日程、聊天记录和过去无数次判断里。谁说过什么，项目卡在哪里，客户为什么不满意，哪些事能交给 AI，哪些事必须自己确认，这些才是工作现场。

但今天大多数人用 AI，还是每次重新把这个现场搭一遍。搭完一次，拆掉一次。下次再问，再从头开始。

所以你会累。
```

## 结尾要求

结尾必须回扣开头场景，不要只总结全文。

结尾句要像观点，而不是口号。示例：

```text
AI 的下一步，不是更会聊天，而是进入你的工作现场。
```

```text
未来人与 AI 的差距，可能不是谁更会提问，而是谁留下了更多值得被理解的自己。
```
