# 写作风格

## 核心定位

这不是传统公众号长文，也不是 AI 新闻摘要，而是适合快速阅读和转发的短观点文章。

文章用 AI 热点、产品细节、历史类比或生活观察作为入口，最后落到人的能力、判断和生活方式。价值不在信息量，而在让读者产生“我以前没这样想过”的感受。

## 标题

标题是一句可以直接传播的观点，简单、有判断，通常包含对比、重新定义或类比。

常用公式：

```text
会做 X 的人 都在 Y
X 时代 请抛弃 A 开始 B
X 好的人 通常不会太差
现在的 X 很像当年的 Y
X 越懂你 越有用
```

不要堆叠专业术语，不要使用夸张的营销标题。

## 正文结构

优先使用四段式：

1. 从一个熟悉的现象或小变化开始。
2. 提出一个反常识解释或重新定义。
3. 用短句、对比、类比或简单等式强化观点。
4. 用一句可以独立传播的话收尾。

## 语言规则

- 每句话只表达一个意思。
- 每段通常 1～3 行。
- 多用“不是……而是……”“以前……现在……”制造对比。
- 少用长段落、复杂解释、抽象术语和连续形容词。
- 可以少量使用英文词、数字和等式，增强视觉记忆点。
- 不追求论证面面俱到，但事实必须准确。
- 不说教，不写空泛鼓励，不使用鸡汤腔。

## 已确认案例

### 会用 AI 的人 都在拆任务

从“把 AI 当搜索框”切入，通过具体提示词对比，重新定义会用 AI 的关键能力：不是更会提问，而是更会拆任务。

### AI 时代 请抛弃 Loading... 开始 Thinking...

从 AI 产品将 `Loading...` 改为 `Thinking...` 的小变化切入，提炼“很多时候你不是卡住了，只是还没想明白”的成长观点。

### 审美好的人 通常不会太差

重新定义审美：不是漂亮，而是一种筛选能力。通过输入如何改变人的语言、气质和判断，落到个人成长。

### 现在的 AI 很像 2010 年智能手机爆发前夜

用智能手机流量类比 Token，用简单等式解释行业趋势：

```text
Token = 流量
大模型 = 芯片
Agent = 手机
Skill = App
```

### AI 越懂你 越有用

从 Agent 更重视工作上下文的趋势切入，落到个人知识沉淀：AI 的价值不仅取决于模型有多聪明，也取决于一个人有没有留下值得被理解的东西。

## 常见失败

| 失败 | 修正 |
|---|---|
| 写成产品发布摘要 | 热点只保留支撑观点所需的最少事实 |
| 一篇文章讲多个观点 | 只保留最有传播力的一句判断 |
| 句子短但内容空 | 加入一个具体观察、对比或类比 |
| 结尾只是总结正文 | 把结尾改成可独立传播的判断 |
| 为了有力量而夸张预测 | 删除无法由来源支持的断言 |
